class sw360_api.SW360(url, token, oauth2=False)¶

Python interface to the Siemens SW360 platform

Authentication against a running SW360 instance is performed using an API token. The token will be sent as HTTP header using the format Authorization: <token_type> <token>. Check your SW360 REST API documentation for details on needed type and how to get the token. token_type is “Bearer” for an OAuth workflow and “Token” for tokens generated via the SW360 UI.

Parameters

  • url (string) – URL of the SW360 instance

  • token (string) – The SW360 REST API token (the cryptic string without “Authorization:” and token_type).

  • oauth2 (boolean) – flag inidacting whether this is an OAuth2 token

close_api()¶

A keep-alive HTTP session is used to access the SW360 REST API. This method allows to explicitely close the connection at a defined time. Normally, you don’t need to call it - session is cleaned up automatically when needed.

Get information about attachments with a given sha1 hash value.

This usually returns zero or one result, but if the same binary file was uploaded and attached to multiple sw360 resources, it will return all matching attachments for the given hash.

API endpoint: GET /attachments?sha1=

update_component_external_id(ext_id_name, ext_id_value, component_id, update_mode='none')¶

Set or update external id of a component. If the id is already set, it will only be changed if update_mode==”overwrite”. The id can be deleted using update_mode==”delete”.

The method will return the old value of the external id or None if it was not set.

API endpoint: PATCH /components

Parameters

  • ext_id_name (string) – name of the external id

  • ext_id_value (string) – value of the external id

  • component_id (string) – the id of the component to be updated

  • update_mode (string) – can be “none” (default), “overwrite” or “delete”

Returns

old value of external id

string
